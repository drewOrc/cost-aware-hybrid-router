# 成本感知混合路由：以等同準確率實現 74% LLM 成本削減

**Bo-Yu Chen**
boyu.chen@my.utsa.edu

---

## 摘要

在多代理 LLM 系統中，使用者的每句話都必須先被「路由」到正確的任務代理。常見做法是用 LLM 來當路由器。這種選擇因其準確率而被接受，但我們想問：這個準確率是否值得這個成本？

我們在 CLINC150 基準測試（150 個意圖、7 個代理 + OOS）上評估四種路由器：(R1) 關鍵詞匹配、(R2) TF-IDF 最近質心、(R3) Claude Haiku 4.5 純 LLM、以及 (R4) 一個混合級聯路由器——用 R1 和 R2 作為信心門控的預過濾器，只在信心不足時才升級給 R3。

在 3 個種子 × 400 筆分層查詢（合計 1,200 筆）下，R4 達到 **82.6% ± 1.2pp**，而 R3 達到 **82.9% ± 0.6pp**。McNemar 配對檢定顯示無顯著差異（p > 0.37，Wilson 95% 信賴區間重疊）。R4 只將 **26.1%** 的查詢送交 LLM，意味著 LLM API 成本降低了 **74.3%**。主要弱點是 OOS（超出範圍）查詢，便宜階段無法可靠地拒絕分類。程式碼、門檻值和逐種子日誌皆已公開。

程式碼：https://github.com/drewOrc/cost-aware-hybrid-router

## 1 引言

在部署的多代理 LLM 系統中，每句使用者輸入都會先經過一個路由器，由它決定要把請求交給哪個下游代理（財務、人資、旅遊等）。業界的常見做法是用像 Claude Haiku 或 GPT-4o-mini 這樣的精簡 LLM 當路由器，把代理清單和 JSON 輸出指令一起餵給它。這種方式方便，而且不怕使用者換不同說法，是 CrewAI、LangGraph 和 AutoGen 的預設方案。但代價是：在生產規模下，每個使用者回合都會產生一次 LLM API 呼叫，累積下來成本可觀。

我們要問的是：這個逐回合的 LLM 呼叫真的有必要嗎？關鍵詞匹配或嵌入向量查找比 API 呼叫便宜三到四個數量級，在本地運算只需毫秒延遲，而且在簡單查詢上可以跟 LLM 一樣準確。如果能用便宜的信心訊號識別出簡單查詢，只把困難查詢送交 LLM，逐回合路由成本就能降低約一個數量級。

關鍵是，這個級聯不需要任何任務專屬的訓練資料：關鍵詞清單來自代理清單，TF-IDF 質心在通用訓練範例上擬合，使得它無需微調即可部署在任何多代理系統中。因此 82.6% 的準確率反映的是零樣本（zero-shot）條件的限制；微調過的分類器會更高，但也需要為每個新領域標註資料，失去了即插即用的優勢。

我們做出四項貢獻：

1. 在統一協議下評估四種路由器：關鍵詞 (R1)、嵌入最近質心 (R2)、Claude Haiku 4.5 純 LLM (R3)、以及混合級聯 (R4)。
2. 進行 3 個種子的論文級評估（種子 42、43、44；合計 1,200 筆），報告 Wilson 95% 信賴區間和 McNemar 配對檢定。
3. 發現 R4 在準確率上與 R3 僅差 0.4pp（82.6% vs 82.9%），同時只將 26% 的查詢升級給 LLM，在 α=0.05 下無顯著準確率損失的情況下降低 74.3% 的 LLM 成本。
4. 證明在信心門控級聯中，便宜階段的**校準**比其獨立準確率更重要：一個 109M 參數的編碼器（76.2%）反而產生比 TF-IDF（74.0%）更差的級聯效果，因為其信心分數校準不佳。

## 2 相關工作

### 意圖分類

CLINC150 基準測試包含 150 個領域內意圖加上一個超出範圍（OOS）類別，專門設計來在分布偏移下測試意圖分類器。強基線包括 SetFit（對比式少樣本微調）和微調的 DeBERTa。我們用 SetFit（compact backbone，17M 參數，每類 16 個範例）作為無 LLM 基線（完整測試集 70.2%），用 Claude Haiku 4.5 零樣本作為成本上限。

### 成本感知 LLM 級聯

FrugalGPT 引入了通用級聯模式（先試便宜模型，信心不足時才升級到貴模型），在單回合 QA 任務上報告高達 98% 的成本降低。Arora et al. 最近在 CLINC150 上評估了 SetFit 和七個 LLM，提出基於不確定性的路由策略。我們的級聯有三個不同之處：(i) 從零參數的關鍵詞階段開始，解決 59% 的查詢；(ii) 便宜階段比較顯示信心校準比獨立準確率更決定級聯效果；(iii) 在代理層級（8 路）而非意圖層級（150 路）運作。

### 學習式路由器

Hybrid LLM 訓練難度預測器節省 40% 大模型呼叫。RouteLLM 用偏好資料訓練達 85%+ 成本降低。AutoMix 用少樣本自我驗證決定是否升級。我們的級聯與這些不同之處在於不需要任何訓練資料，信心訊號來自關鍵詞和嵌入階段本身。

### 代理路由系統

CrewAI、LangGraph 和 AutoGen 都預設用 LLM supervisor 做意圖路由。Semantic Router（Aurelio Labs）走相反路線，完全透過嵌入相似度路由。我們提供了在 CLINC150 上的量化測量：非 LLM 階段能吸收多少路由工作量，以及準確率代價是多少。

## 3 方法

### 3.1 資料集與標籤空間

使用 CLINC150 測試集（5,500 筆查詢，含 1,000 筆 OOS）。將 150 個細粒度意圖摺疊為 7 個任務級代理（auto、device、finance、kitchen、meta、productivity、travel）加 OOS 類別，形成 8 路決策問題。

### 3.2 四種路由器

**R1: 關鍵詞路由器。** 為每個目標代理建立關鍵詞清單。分數是匹配的關鍵詞權重總和，預測標籤是代理間的 argmax。

**R2: TF-IDF 最近質心路由器。** 在 CLINC150 訓練集上擬合 TF-IDF 向量器（10K 特徵，unigram + bigram，sublinear TF），計算每個代理的質心。推論時，查詢向量化後與所有質心比較餘弦相似度。選擇 TF-IDF 而非神經句編碼器，是因為其低量級信心分數對級聯的信心門控機制有更好的校準。

**R3: LLM 路由器。** 使用 Claude Haiku 4.5，帶有列出 8 個目標代理及描述的簡短系統提示。溫度為 0；每筆查詢是單次 API 呼叫，無上下文範例。

**R4: 混合級聯。** 在 R1 → R2 → R3 上的信心門控級聯，帶兩個門檻值（kt, et）：
- 如果 R1 信心 ≥ kt → 用 R1
- 如果 R2 信心 ≥ et → 用 R2
- 否則 → 升級給 R3（LLM）

### 3.3 門檻值調校

在 CLINC150 驗證集（3,100 筆，與測試集分開）上透過網格搜索調校。選擇在準確率 ≥ 88% 的約束下最小化 LLM 呼叫率的配置。最終值：kt = 0.5, et = 0.10（驗證集上 22.7% LLM 呼叫率）。

### 3.4 評估協議

LLM 評估使用每個種子 400 筆分層子樣本（每類 50 筆 × 8 類）以控制 API 費用。使用 3 個種子（42, 43, 44）。報告 Wilson 95% 信賴區間和 McNemar 精確二項配對檢定。

### 3.5 成本核算

R3 和 R4 的 API 成本從 Anthropic 回應 metadata 直接測量。全部 3 種子實驗的總 API 成本為 **$0.44**。

## 4 結果

### 4.1 主要結果

| 路由器 | 準確率 | Wilson CI | LLM 呼叫率 | 每筆成本 |
|--------|--------|-----------|-----------|---------|
| R1 關鍵詞 | 64.8% | --- | 0 | ~0 |
| R2 嵌入 | 74.0% | --- | 0 | ~0 |
| SetFit | 70.2% | --- | 0 | ~0 |
| R4 無LLM | 74.9% | --- | 0 | ~0 |
| **R3 LLM** | **82.9%** | [80.7, 84.9] | 100% | 2.9e-4 |
| **R4+LLM** | **82.6%** | [80.3, 84.6] | 26% | 7.5e-5 |

R3 和 R4+LLM 的 95% 信賴區間重疊。沒有 LLM 後備（R4 無LLM），準確率降至 74.9%——LLM 在它處理的 26% 查詢上確實發揮作用，而剩餘 74% 由便宜階段以等同準確率解決。

### 4.2 統計顯著性

McNemar 配對檢定（每個種子 n=400）：

| 種子 | Δ (R4−R3) | McNemar p | 顯著？ |
|------|-----------|-----------|--------|
| 42 | +0.00pp | 1.000 | 否 |
| 43 | −1.75pp | 0.371 | 否 |
| 44 | +0.75pp | 0.755 | 否 |

三個種子的差異皆不顯著。合計 Cohen's h = 0.009（可忽略效果）。

### 4.3 成本節省

R4+LLM 跨種子的平均 LLM 呼叫率為 26.1% ± 1.7pp，整體 **LLM 成本降低 74.3%**。投射到 1,000 筆查詢：R4+LLM 花 $0.075 vs R3 的 $0.293。以生產量（每天 1 萬筆查詢）計算，每年約節省 ~$800；若用更貴的路由器（如 Claude Sonnet，Haiku 定價的 ~15 倍），同樣 74% 的降低每年可省 ~$12K。

### 4.4 定性分析

三個級聯階段分配種子 42 的查詢分布：**59.3% 在 R1 停止**（11.0% 錯誤率）、**17.0% 在 R2 停止**（16.2% 錯誤率）、**23.8% 升級到 R3**。

OOS 仍是最弱環節：級聯的 OOS 準確率為 48.0% vs R3 的 58.0%（Δ = −10pp）。在領域內查詢上，級聯與 R3 持平甚至略高（87.5% vs 86.5%）。準確率代價完全集中在 OOS 處理上。

### 4.5 門檻值敏感度

級聯提供一個平滑的準確率—成本旋鈕：實踐者可以透過調整 α 或準確率下限來選擇操作點。

### 4.6 透過 Platt Scaling 的校準路由

原始門檻值 kt 和 et 在不同量尺上運作。替代方案是透過 Platt scaling（邏輯迴歸）將兩者校準為 P(correct)，然後用單一門檻 τ。

| 方法 | 便宜階段準確率 | 預期含LLM準確率 | LLM 呼叫率 |
|------|-------------|---------------|-----------|
| 網格搜索 (kt=0.5, et=0.1) | 74.4% | 83.0% | 26.0% |
| 校準 (τ=0.75) | 75.2% | 83.9% | 25.4% |

McNemar p < 0.001，但 Cohen's h = 0.019（可忽略效果量）。主要優勢是可解釋性：單一的 τ = 0.75（「當 P(correct) < 75% 時升級」）取代了兩個資料集專屬的數值。

### 4.7 替代便宜階段

| R2 模型 | 參數量 | R2 準確率 | 級聯準確率 | LLM 呼叫率 |
|---------|--------|----------|----------|-----------|
| TF-IDF（我們的） | 10K 特徵 | 74.0% | **86.7%** | 26.0% |
| all-MiniLM-L6-v2 | 22M | 73.3% | 75.6% | 1.4% |
| all-mpnet-base-v2 | 109M | **76.2%** | 77.2% | 1.3% |

反直覺地，獨立最強的模型（MPNet, 76.2%）產生最差的級聯。原因是信心校準：神經編碼器即使對錯分類的查詢也給出高相似度（>0.3），gate 幾乎不觸發。TF-IDF 的低量級相似度讓門檻有真正的辨別力。在信心門控級聯中，**便宜階段的校準比其原始準確率更重要**。

## 5 討論

**級聯何時有效？** 級聯之所以有效，是因為 CLINC150 的難度分布是雙峰的。大多數領域內查詢有明顯的關鍵詞，R1 或 R2 就能處理。困難的尾部（主要是 meta 意圖和 OOS）需要 LLM。

**為什麼不微調？** 微調分類器（如 SetFit 或 DeBERTa）在 CLINC150 的 8 路任務上可能超過 90%。但代價是部署摩擦：需要為每個目標代理標註範例、代理清單變動時重新訓練、還需要模型服務端點。級聯不需要這些——它從代理清單本身（關鍵詞清單、TF-IDF 質心）衍生信心訊號，代理增減時即插即用。

**延伸到多回合任務。** 單回合路由只是全貌的一部分。在多回合代理任務中，同樣的想法適用於不同粒度：用便宜模型**拆解**任務，保留貴模型來**執行**。我們在 τ-bench 上的另一篇工作測試了這個想法。

---

## 6 結論

在 CLINC150 上，級聯在 0.4pp 內匹配純 LLM 路由器的準確率，同時削減 74% 的 LLM 呼叫。實際意義是：LLM 路由器只在查詢的困難尾部才必要，大約佔流量的四分之一，而截斷點可以用團隊自己的資料和我們釋出的程式碼來測量。
